Properties Class

Defined in: js\classes\Properties.js:5
Module: Properties

A general Property class. If the argument is an object, then its properties are copied into Property instance. If the argument is a string, then it will be splitted according to the pattern "key: value;" to populate object properties.

toggleProperty

(
  • propName
  • val
  • altVal
)
Void

Defined in js\classes\Properties.js:325

Switches property propName between val and altVal in the following way:

  1. if the instance has property propName equal to val, then the method imposes its value to be altVal (if altVal is not defined, then the key propName gets dropped).
  2. if the instance contains property propName equal to altVal, then the method sets its value to be val.
  3. if the instance does not have property propName, then the method imposes it to be val.

Parameters:

  • propName String

    property key to toggle (i.e., "width", "position")

  • val String

    value of property key (i.e., "25em", "fixed")

  • altVal String | Null

    alternative value of property key (i.e. null, "0em"). This parameter is optional.

Returns:

Void:

toString

() String

Defined in js\classes\Properties.js:519

Available since 0.0.5

String representation of the instance. If _mode is set to 0, the representation of the following type is produced:

width="20" title="read me!"
If _mode is set to 1, the representation of the following type is produced:

width: 50px; color: red
It parses all core keys and if

  1. corresponding value responds to a toString() method
  2. corresponding value is a number and _mode is 1, then 'px' is appended (resolve this explicit dependency on unit of measurement!)
  3. corresponding value is a string
then uses it for further substitution. Otherwise, the value is ignored.

Further, if the corresponding string representation of the value turns out to be empty or undefined, the whole key-value pair is ignored.

Returns:

String:
