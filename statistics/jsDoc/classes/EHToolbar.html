highlight

(
  • ed
  • prop
  • elemId
)
Void

Defined in js\eventHandlers\EHToolbar.js:53

Switches the state of element with id elemId to be on if the cursor is located inside element that has a property descibed in object prop, otherwise the state is set to off.

on/off states are given by CKEDITOR constants TRISTATE_ON and TRISTATE_OFF.

Object prop is required to have the following keys (presence of other keys is allowed):

name

name of the attribute to search among start element ascendants

value

value that the attribute is supposed to have

Uses setState method with hardcoded optional parameter set to to 'cke_button'.

Parameters:

  • ed CKEDITOR.editor

    editor instance

  • prop Object

    name of the attribute to search

  • elemId String

    id of the element which state should be changed

Boolean

Defined in js\eventHandlers\EHToolbar.js:100

Returns true if event corresponds to pressing one of the arrow keys: 'Left', 'Right', 'Down', 'Up'. The check is performed by the code of the pressed key: 37 for 'Left', 39 for 'Right', 40 for 'Down', 38 for 'Up'.

Void

Defined in js\eventHandlers\EHToolbar.js:14

Attaches listeners for two type of events on ed element:

mousedown event
triggers when one moves cursor by clicking mouse button.
keydown event
triggers when one moves arrow keys. Uses isArrow method in order to decide whether the event should be handled or not.

Event handler for the above events is highlight.

    CKEDITOR.editor instance
